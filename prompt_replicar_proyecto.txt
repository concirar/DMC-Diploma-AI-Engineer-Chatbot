  - Desbalance original: ~19:1 (no quiebra vs quiebra)

3.4 PREPROCESAMIENTO
------------------------------------------------------------------------

  Paso 1 — Eliminar columnas con > 50% de nulos:
    threshold = 0.50
    cols_to_drop = missing_pct[missing_pct > threshold * 100].index.tolist()
    X_clean = X.drop(columns=cols_to_drop)
    (En este dataset, ninguna columna supera el umbral, por lo que X_clean == X)

  Paso 2 — Imputación con mediana:
    imputer = SimpleImputer(strategy='median')
    X_imputed = pd.DataFrame(imputer.fit_transform(X_clean), columns=X_clean.columns)

  Paso 3 — Clip de outliers (percentil 1 y 99):
    for col in X_imputed.columns:
        q01 = X_imputed[col].quantile(0.01)
        q99 = X_imputed[col].quantile(0.99)
        X_imputed[col] = X_imputed[col].clip(q01, q99)

  Paso 4 — Train/Test split estratificado:
    X_train, X_test, y_train, y_test = train_test_split(
        X_imputed, y_series,
        test_size=0.20,
        random_state=42,
        stratify=y_series
    )
    # Resultado: Train (34724, 65) | Test (8681, 65)
    # Train positivos: 1673 (4.82%) | Test positivos: 418 (4.82%)

  Paso 5 — Balanceo con SMOTE (solo sobre training set):
    smote = SMOTE(random_state=42)
    X_train_bal, y_train_bal = smote.fit_resample(X_train, y_train)
    # Resultado: (66102, 65) — 33051 de cada clase

    - Calcula SHAP values para la empresa

================================================================================
NOTAS IMPORTANTES
================================================================================

  1. El split train/test usa random_state=42. Si se usa cualquier otro valor,
     los IDs del CSV no coincidirán con las filas del modelo. No cambiar.

  2. SMOTE se aplica SOLO al training set, nunca al test set. El test set
     mantiene el desbalance original para una evaluación realista.

  3. scale_pos_weight=1 en XGBoost porque SMOTE ya balanceó la clase.
     Si se elimina SMOTE, usar scale_pos_weight=19 (ratio del desbalance original).

  4. El SHAP explainer (TreeExplainer) corre localmente en Streamlit con el
     modelo cargado vía mlflow.xgboost.load_model(). La API REST de MLflow
     (puerto 5002) es opcional y solo devuelve la clase (0/1), no probabilidades.

  5. Para obtener probabilidades se usa model.predict_proba() directamente
     sobre el modelo cargado en memoria, no via la API REST.

  6. El modelo Ollama (gemma4:e2b) debe estar descargado antes de ejecutar la app:
     ollama pull gemma4:e2b

  7. Si gemma4:e2b no está disponible, puede reemplazarse por cualquier modelo
     Ollama compatible (gemma2:2b, llama3.2:3b, qwen2.5:3b, etc.).
     Solo cambiar la constante OLLAMA_MODEL en app.py.
